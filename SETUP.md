# Quick Setup for chrisdaley.biz Root Domain

## What you need to do RIGHT NOW:

### 1. Rename this repository
- Go to: https://github.com/chebizarro/sovereign-systems-lab/settings
- Scroll to "Repository name"
- Change from `sovereign-systems-lab` to `chebizarro.github.io`
- Click "Rename"

### 2. Configure GitHub Pages
- Go to: https://github.com/chebizarro/chebizarro.github.io/settings/pages
- Under "Custom domain", enter: `chrisdaley.biz`
- Click "Save"
- Check "Enforce HTTPS" (after DNS propagates)

### 3. Update your DNS (at your domain registrar)
Replace existing records with:
```
Type   Name   Value
----   ----   -----
A      @      185.199.108.153
A      @      185.199.109.153
A      @      185.199.110.153
A      @      185.199.111.153
CNAME  www    chebizarro.github.io
```

### 4. Push to deploy
```bash
git add .
git commit -m "Configure for root domain deployment"
git push origin main
```

## What happens next:

1. GitHub Actions builds the site (check: https://github.com/chebizarro/chebizarro.github.io/actions)
2. DNS propagates (1-48 hours, usually ~1 hour)
3. Site goes live at https://chrisdaley.biz/

## Important notes:

- **Only ONE repo per account can be the user site** (root domain)
- Your other repo will need to move to a subdomain or different path
- The CNAME file is auto-generated by the workflow
- All URLs are already configured for chrisdaley.biz (no code changes needed)

## Verify it worked:

After DNS propagates, check:
- https://chrisdaley.biz/ (homepage)
- https://chrisdaley.biz/rss.xml (RSS feed)
- https://chrisdaley.biz/sitemap.xml (sitemap)

See `.github/DEPLOYMENT.md` for full technical details.
