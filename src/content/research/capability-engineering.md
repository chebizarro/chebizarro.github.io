For a long time, engineering meant making machines do more per unit of
input. *Capability engineering* is the same discipline turned toward
people: making a competent person effective, and an effective person
formidable, using the tools now available.

## The unit of work

Not the task. The *capability envelope*: what one motivated person can
do, end-to-end, in a week, without asking permission.

Every serious tool of the last two decades — Git, the modern CLI, the
web browser, the smartphone, the code agent — expanded that envelope.
The interesting engineering question is what expands it next.
